DocFlow — 文档托管 + 版本归档系统

# DocFlow — 文档托管 + 版本归档


> **项目名称**：DocFlow

> **实现路径**：FastAPI + 纯文件(Markdown)存储 + 原生 HTML 前端；读取公开 / 写入 Bearer token；Agent 通过 POST 推送，自动归档历史版本，前端默认显示当前版、右侧时间线回看历史。无数据库，备份即打包 `data/`。


一个极简服务：文档以 Markdown 存在服务器，**读取公开**（点开就能看），**写入需 token**（给 Agent 用）。

每次写入自动归档旧版本，前端默认显示最新版，右侧时间线可加载任意历史版本。


## 现状（占位）


项目已预置 4 个占位文档，页面打开就有内容，用 Agent 推送真实内容覆盖即可：

- 个人简历 · 中文版 v4.0 / Resume · English v2.0（分类：个人简历）

- AI 记忆系统 · Claude / Gemini（分类：AI 记忆系统）


加新文档不用改代码——POST 一个新 id，索引页自动出现。索引页地址 `/`，单文档形如 `/d/resume-cn`。


## 部署注意


- 默认端口 **8000**。你 VPS 已跑 OpenWebUI，启动前确认端口不冲突，要改就改 `--port`。

- 读取默认公开，链接拿到就能看。简历这类要私密的，按文末「读取也私密」处理后再挂公网。



## 设计要点（先看这个）


- **内容与展示分离**：你/Agent 只管推 Markdown，前端样式和版本逻辑不用碰。

- **版本即文件**：`data/docs/<id>/<version>.md` + 一个 `meta.json` 索引。没有数据库，备份就是打包 `data/` 目录。

- **当前版本动态决定**：每次 POST 默认把新版本设为「当前」；前端永远先显示当前版本。

- **历史只读不删**：旧版本文件保留，时间线随时回看。


## 目录结构


docflow/
├── app.py            后端（FastAPI）
├── requirements.txt
├── seed.py           初始化 4 个占位文档（跑一次）
├── agent_client.py   Agent 推送示例
├── static/
│   ├── index.html    上层索引页（按分类聚合）
│   └── doc.html       文档查看页（渲染 + 历史时间线）
└── data/docs/        文档数据（运行后生成）


## 本地 / VPS 启动


pip install -r requirements.txt
export DOCFLOW_TOKEN="你自己的长随机串"      # 必改，写入凭证
python seed.py                              # 可选：先放占位文档
uvicorn app:app --host 0.0.0.0 --port 8000

打开 `http://<服务器IP>:8000/` 即索引页。单文档链接形如 `http://<IP>:8000/d/resume-cn`。


### 挂成常驻服务（systemd）


`/etc/systemd/system/docflow.service`：


[Unit]
Description=DocFlow
After=network.target

[Service]
WorkingDirectory=/root/docflow
Environment=DOCFLOW_TOKEN=你自己的长随机串
ExecStart=/usr/bin/uvicorn app:app --host 0.0.0.0 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target

systemctl daemon-reload && systemctl enable --now docflow

### 可选：Nginx 反代 + HTTPS


你 VPS 已跑 OpenWebUI，端口别撞即可。Nginx 片段：


location /docs/ {
    proxy_pass http://127.0.0.1:8000/;
    proxy_set_header Host $host;
}

证书用 certbot 自动签。注意：读取是公开的，链接拿到就能看，**别把敏感简历放公网无鉴权**——要私密就给读取也加一层（见下）。



## Agent 接入


### 接口表


| 方法 | 路径 | 鉴权 | 说明 |

|---|---|---|---|

| GET | `/api/docs` | 否 | 所有文档元数据 |

| GET | `/api/docs/{id}` | 否 | 单文档元数据（含版本时间线） |

| GET | `/api/docs/{id}/raw?version=x` | 否 | 某版本 Markdown 原文（缺省取当前） |

| POST | `/api/docs/{id}` | **是** | 创建/更新一个版本 |


### POST 字段


| 字段 | 必填 | 说明 |

|---|---|---|

| `content` | ✓ | Markdown 正文 |

| `version` | ✓ | 版本号，如 `4.0` |

| `title` | 首次建议 | 文档标题 |

| `category` | 首次建议 | 分类（决定索引页分组） |

| `note` | | 本次更新说明，显示在时间线 |

| `set_current` | | 默认 `true`；补录旧版本时传 `false` |


### curl 示例


curl -X POST http://127.0.0.1:8000/api/docs/resume-cn \
  -H "Authorization: Bearer 你的TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content":"# 我的简历\n\n...", "version":"4.0",
       "title":"个人简历 · 中文版", "category":"个人简历",
       "note":"更新工作经历"}'

### Python（Agent 直接用）


见 `agent_client.py`，核心就一个 `push()` 函数：


from agent_client import push
push("resume-cn", md_text, "4.0",
     title="个人简历 · 中文版", category="个人简历", note="更新")


## 常见操作


- **加新文档**：直接 POST 一个新 `id` 即可，索引页自动出现，无需改代码。

- **改分类/标题**：POST 时带新的 `category`/`title`，会覆盖元数据。

- **回滚**：把旧版本内容用新版本号再 POST 一次即可（旧版本始终在）。

- **备份**：`tar czf backup.tgz data/`。

- **改 token**：改环境变量重启服务。


## 如果要读取也私密


`app.py` 里给 GET 接口和页面路由加上和 POST 一样的 `check_token`，前端 fetch 时带上 `Authorization` 头（或改成短期签名链接）。当前默认是公开读取，符合「点开就能看」。
