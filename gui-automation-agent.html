视觉驱动的 GUI 自动化与智能筛选 Agent

独立开发 · 2026
基于多模态视觉模型实时读屏的移动端 GUI 自动化框架：不依赖元素 ID 或 DOM 结构，纯视觉识别 + 坐标操作，配合 LLM 语义打分对列表型任务进行自动筛选与处理。具备行为拟真、异常自恢复、无人值守长跑能力。

一、背景与目标

移动端 App 的自动化处理长期面临三个工程难题：

  1. UI 无稳定接口——多数 App 不暴露 API，原生 UI 频繁改版，依赖元素 ID / 控件树的传统自动化极易失效；
  2. 操作拟真度要求高——机械、规律的操作节奏与轨迹形态，在交互层面与真人差异明显，稳定性差；
  3. 列表型任务缺乏筛选——面对大量同构条目（如信息流、列表项），缺乏语义判断会导致无差别处理、有效产出率低。

本项目目标：构建一个视觉驱动、语义筛选、可无人值守长跑的移动端 GUI 自动化框架，把"看屏幕—做判断—执行操作"的人工链路完整自动化。

三、核心技术实现

1. 视觉驱动的 GUI 自动化

  • 使用字节跳动 豆包 Seed1.5-VL 多模态视觉模型实时截图读屏——该模型针对 GUI 导航与 Agent 操作场景做过专门优化，能直接从截图定位可交互元素；
  • 不依赖元素 ID 或 DOM 结构，因此天然适配 App 的 UI 改版，鲁棒性远高于控件树方案；
  • ADB 层为纯截图 + 坐标点击，不注入任何 hook，操作链路与真实用户路径一致；
  • 广告弹窗、权限申请等异常弹层自动识别并关闭，流程不中断。

2. 贝塞尔曲线轨迹拟真

滑动轨迹 = 贝塞尔曲线(起点, 随机控制点, 终点)
  ├─ 控制点在合理范围内随机采样
  ├─ 曲线采样点数 = f(距离) · 随机抖动
  └─ 每段操作附加随机延迟 ±offset

直线、等速、零抖动的操作轨迹与真人交互差异显著。引入贝塞尔曲线后，滑动路径在空间形态与时序节奏上均接近真人手势，是 GUI 自动化中提升交互拟真度的成熟技法。

3. LLM 语义匹配打分引擎

  • 将核心画像与关键词构建为候选画像，注入系统提示作为评分基准；
  • 每个列表条目进入详情页后，Seed1.5-VL 截图提取文本，交由 LLM 进行 0–100 分语义评分；
  • 匹配度 ≥ 设定阈值 才执行处理，低于阈值直接跳过；
  • 用语义理解替代关键词硬匹配，显著降低误处理率。

工程亮点：节拍即副产品

进入条目 → 页面加载 → Seed1.5-VL 截图识别
    → LLM 语义打分（耗时 1-3s）
        ├─ ≥ 阈值 → 处理 → 返回列表
        └─ < 阈值 → 跳过 → 返回列表
            └─ 操作间隔由推理耗时自然构成

关键设计：LLM 评分本身的推理耗时，天然构成了操作之间的节拍，无需硬编码 sleep。这让系统的吞吐节奏与稳定性自然平衡——既避免了固定延时带来的机械感，也省去了人工调参间隔。一个为"判断"而存在的环节，顺带解决了"节奏"问题，是流程设计上的一处巧合收益。

4. 多层异常处理

异常类型处理策略

六、能力迁移

该框架与具体业务场景解耦，可迁移至任意"面对大量同构 GUI 条目、需逐条判断并执行操作"的移动端场景：信息流自动筛选、批量内容审核辅助、UI 回归测试、无 API 应用的数据采集与录入等。视觉驱动 + 语义筛选这一组合，本质上是把"人盯着屏幕做重复判断"的工作整体外包给了 Agent。
